Components to the Reg System by Directory:

api: Future API placeholder

apiDocs: documentation placeholder for the API

atcon: On site registration system
    css: css files specific to atcon
    csslib: symlink to common css files (project wide)
    images: local images
    js: client side javascript for atcon pages, named by page involved
    jslib: symlink to common javascript libs (project wide)
    lib: common routines for atcon usage, not used by other modules
    scripts: server side functions called by javascript via ajax

Composer: composer.json and composer.lock
    vendor: untracked location of the composer output of the vendor tree

config: text configuration files

config-sample: prototypes of the configuration files for copying to config and building your configuration

controll: Admin back end to ConTroll Reg System
    css: css files specific to controll
    csslib: symlink to common css files (project wide)
    js: client side javascript for atcon pages, named by page involved
    jslib: symlink to common javascript libs (project wide)
    images: local images
    lib: common functions for controll
    md: end user help files in markdown notation
    reports: server side functions that produce reports
    scripts: server side functions called by javascript via ajax

crons: shell scripts of jobs to schedule

crons-sample; prototypes of the scheduled jobs

csslib: master copy of csslib, where the symlinks point to

install: scripts and files for installing ConTroll

lib: system wide functions (used by all subsystems)

onlinereg: web based registration
    css: css files specific to atcon
    csslib: symlink to common css files (project wide)
    images: local images
    javascript: client side javascript
    jslib: symlink to common javascript libs (project wide)
    lib: common functions used by onlinereg

portal: registration portal (web based)
    css: css files specific to portal
    csslib: symlink to common css files (project wide)
    js: client side javascript for atcon pages, named by page involved
    jslib: symlink to common javascript libs (project wide)
    images: local images
    lib: common functions for portal
    scripts: server side functions called by javascript via ajax

regapi: Future API placeholder

scripts: global scripts used by the system

scripts-sample: prototype scripts to copy

System-Documents-and-Coding-Standarts: internal documentation including recent ER diagram

UserDoc: end user documents

UserDoc-Samples: end user documents from various conventions or our own as samples

vendor: exhibitor registration, used by two websites artist and vendor right now
